nativemodel

Identifier:
eu.hydrologis.jgrass.models.eu.hydrologis.jgrass.models.nativemodel

Since:
[Enter the first release in which this extension point appears.]

Description:
This extention point creates the necessary wraps and operating system tweaks around the native GRASS commands, making them so executable from within the JGrass Console Engine.

Configuration Markup:

<!ELEMENT extension (nativemodel*)+>

<!ATTLIST extension

point CDATA #REQUIRED

id    CDATA #IMPLIED

name  CDATA #IMPLIED

>


<!ELEMENT nativemodel (EMPTY?)>

<!ATTLIST nativemodel

commandname CDATA #REQUIRED

guixml      CDATA #IMPLIED

>

  • commandname -
  • guixml -

    A correct xml file content that can represent the command as off the uibuilder guidelines. This string will be correctly placed in the system as a file and taken as gui definition for the command.

    For example for h.pitfiller the gui definition is:

    <command descr="h.pitfiller" name="h.pitfiller">
        <field desc="in" name="input" order="0" repr="--igrass-elevation #" required="true" type="rastermap"/>
        <field desc="out" name="output" order="1" repr="--ograss-pit #" required="true" type="string"/>
        <i18n key="in" en="input elevation map" it="mappa di elevazione in ingresso"/>
        <i18n key="out" en="depitted output map" it="mappa depittata in uscita"/>
    </command>
